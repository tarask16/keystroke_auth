## 12. Baseline conclusion

The current baseline confirms that a simple MLP can classify users by fixed-text keystroke dynamics with usable but not production-grade authentication quality.

Key operating point:

- authentication threshold: `0.108000`;
- target FAR: `1.00%`;
- empirical FAR: `1.00%`;
- empirical FRR: `5.54%`.

Main limitations:

- softmax probability is used as an authentication score;
- no user-specific thresholds are used yet;
- the baseline is closed-set and does not model unknown users explicitly;
- false rejection is still visible and should be reduced in later experiments.

Recommended next steps:

- compare deeper MLP architectures;
- add user-specific thresholds;
- evaluate embedding-based verification;
- add per-user FAR/FRR diagnostics;
- test robustness across sessions and acquisition conditions.
